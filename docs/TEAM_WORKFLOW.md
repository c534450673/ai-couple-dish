# AI Couple Dish - 三人团队协作工作计划

## 团队成员

### 1. 情侣A（小明）- 测试体验者
- 角色：体验用户A，模拟真实情侣中的"发起方"
- 职责：全流程体验，记录所有异常和体验问题
- 测试重点：注册/登录 → 生成情侣码 → 投喂 → 添加菜单 → 心愿单 → 纪念日 → 笔记

### 2. 情侣B（小红）- 测试体验者
- 角色：体验用户B，模拟真实情侣中的"接受方"
- 职责：全流程体验，与情侣A配合完成双人交互场景
- 测试重点：注册/登录 → 使用情侣码绑定 → 接收投喂 → 查看菜单 → 心愿实现 → 通知接收

### 3. 资深全栈工程师（老王）- TDD开发
- 角色：修复所有发现的问题
- 职责：按照TDD方式（先写测试→再修复→测试通过）修复bug
- 约束：每个修复必须先写failing test，确认测试失败，再修复代码使测试通过

## 循环执行机制

```
┌─────────────────────────────────┐
│   情侣A+情侣B 全流程体验        │
│   → 记录所有Bug到 Bug List      │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│   Bug List 为空？               │
│   YES → 🎉 完成所有修复！       │
│   NO  → 继续下一步             │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│   全栈工程师 TDD 修复           │
│   1. 先写 failing test          │
│   2. 确认测试失败               │
│   3. 修复代码                   │
│   4. 确认测试通过               │
│   5. 运行全部测试确保无回归      │
└──────────────┬──────────────────┘
               │
               ▼
┌─────────────────────────────────┐
│   情侣A+情侣B 回归验证          │
│   → 修复正确？删除对应Bug       │
│   → 修复不正确？标注异常点      │
│     → 工程师继续修复            │
└──────────────┬──────────────────┘
               │
               ▼
          回到第一步
```

```
  阶段: X.X

  标题: 简述
  复现步骤: 
    1. ...
    2. ...
  预期结果: ...
  实际结果: ...
  修复状态: 待修复/修复中/待验证/已关闭/重新打开
  修复者: 全栈工程师
  TDD测试: 测试文件路径
```
