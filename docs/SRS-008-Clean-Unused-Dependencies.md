# SRS-008: 清理未使用的依赖

## 1. 需求背景

`pom.xml` 中可能引入了未使用的依赖，增加了：
- 项目构建体积
- 潜在的安全漏洞（依赖中的漏洞）
- 维护成本（需要关注依赖更新）

## 2. 当前问题

### 2.1 疑似未使用的依赖

**位置**: `backend/pom.xml`

根据代码扫描，发现以下依赖可能未使用：

| 依赖 | 用途推测 | 使用情况 |
|------|----------|----------|
| `wx-java-miniapp` | 微信小程序 SDK | 未发现使用代码 |
| `weixin-java-mp` | 微信公众号 SDK | 未发现使用代码 |
| 部分 Hutool 工具 | 工具库 | 部分未使用 |

### 2.2 验证方法

1. IDE 依赖分析
2. 代码搜索确认
3. 运行时日志确认无初始化错误

## 3. 解决方案

### 3.1 执行依赖分析

```bash
# 使用 Maven Dependency Analyzer
mvn dependency:analyze

# 分析结果
[INFO] --- maven-dependency-analyzer:1.12.0:analyze (default-cli) @ backend ---
[WARNING] Used undeclared dependency: ...
[WARNING] Unused declared dependency: ...
```

### 3.2 逐个确认

对每个"未使用"的依赖：
1. 搜索代码中是否有 import
2. 检查是否有运行时反射使用
3. 确认 CI/CD 中是否有特殊用途
4. 咨询团队成员

### 3.3 清理步骤

1. 注释掉疑似未使用的依赖
2. 运行单元测试确保通过
3. 运行集成测试确保通过
4. 确认无运行时错误
5. 删除依赖声明

## 4. 验收标准

- [ ] 执行 `mvn dependency:analyze`
- [ ] 确认每个未使用依赖的状态（删除/保留/备注原因）
- [ ] 清理确认无用的依赖
- [ ] 单元测试全部通过
- [ ] 集成测试全部通过
- [ ] 更新 `pom.xml` 注释说明保留原因（如有保留）

## 5. 影响范围

- `backend/pom.xml`

## 6. 优先级

**P1** - 中优先级

## 7. 预计工时

2-4 小时

## 8. 注意事项

- 部分依赖可能是通过传递依赖引入的，删除顶层依赖可能不影响
- 某些依赖可能在测试代码中使用但主代码未使用
- Spring Boot Starter 的传递依赖不要轻易删除
