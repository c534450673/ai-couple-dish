## 一、文档概述

### 1.1 背景
「情侣私密菜单」是一款面向情侣的美食记录应用，核心功能包括餐厅收藏、投喂互动、纪念日管理、心愿单等。经过对现有功能的全面分析，本PRD旨在识别产品优化方向，提升用户体验和产品竞争力。

### 1.2 目标
- 提升用户留存率和活跃度
- 增强情侣互动体验
- 完善核心功能闭环
- 探索商业化可能性

## 二、核心功能优化

### 2.1 用户登录与注册模块

#### 2.1.1 现状问题
| 问题 | 严重程度 | 影响范围 |
|------|----------|----------|
| 微信登录仅在小程序端可用，H5端提示"请在微信中打开" | 高 | H5用户 |
| 验证码发送后无实际短信服务，仅开发环境可用 | 高 | 所有新用户 |
| 用户协议和隐私政策内容为空 | 中 | 合规风险 |
| Apple登录提示"开发中" | 中 | iOS用户 |

#### 2.1.2 优化方案

**P0 - 必须修复**
```
需求ID: USR-001
需求名称: 完善登录认证体系
优先级: P0

详细描述:
1. 接入短信服务商（如阿里云短信、腾讯云短信）
   - 配置短信模板
   - 实现发送频率限制（已有60秒限制）
   - 添加发送日志记录

2. 完善用户协议和隐私政策
   - 委托法务起草正式文档
   - 添加版本号和生效日期
   - 支持"阅读全文"弹窗

3. 微信开放平台接入
   - H5端接入微信网页授权
   - 支持微信扫码登录

验收标准:
- 真实手机号可收到验证码
- 用户协议内容完整可见
- H5端可通过微信授权登录
```

**P1 - 建议实现**
```
需求ID: USR-002
需求名称: 一键登录优化
优先级: P1

详细描述:
1. 本机号码一键登录（阿里号码认证服务）
2. 登录态持久化优化（7天免登录）
3. 多设备登录互踢提醒
```

| 投喂过期后无提醒 | 中 | 用户错过投喂 |
| 无法撤回投喂 | 低 | 误操作无法纠正 |

#### 2.4.2 优化方案

**P1 - 建议实现**
```
需求ID: FED-001
需求名称: 投喂机制优化
优先级: P1

详细描述:
1. 投喂次数调整

4. 投喂记录
   - 历史投喂日历视图
   - 投喂统计（发送/接收/接受率）
```
